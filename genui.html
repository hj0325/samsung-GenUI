Hierarchy
Visual grammar for the 4+2+1 orchestration. Each rule maps a state-packet field to how it should look on canvas.
Classification → Path A Layout Recipe 4 patterns
◎ 몰입 보호형 (Focus Protection) → vertical-stack · 1 primary-task with dominant subject + minimal state, supporting collapsed
◆ 맥락 재구성형 (Context Reconstruction) → grid or vertical-stack · multiple supporting groups of context tiles (home widgets, notification shade, QS panel)
→ 흐름 연속형 (Flow Continuity) → vertical-stack · primary-task with subject + state + action grouped, plus optional supporting context
⟨⟩ 다자 협업형 (Multi-party Coordination) → overlay-stack · system-dialog overlay group on top of underlying surface (handoff, share, confirm)
Path A's classifier (heuristic) tags scenarios with these patterns; the composer LLM is free to deviate when the scenario warrants. Container + group-role + child-role choices encode the pattern in canonical schema, NOT a named template.
Attention Mode → Density 3 modes
glanceable → density=compressed; max 3 items, larger type, wide whitespace
distracted → density=normal; default gap, mid type scale
focused → density=expanded; compact gap, denser grid, many items
Path A's interpreter emits attentionMode + densityMode as separate enums; this panel shows the typical pairing rule.
Urgency → Accent Color 3 rules
low → neutral surface
medium → Samsung blue accent on hero
high → warning red ring on must_show
Component Role → Visual Treatment 7 roles
chrome → status bar, app bar, gesture bar — flush to frame, no card decoration
subject → primary artifact (recipe step, message body, song) — dominant typography (19px bold), larger card padding
state → timer, progress, step counter — attached to subject (negative top margin), 13px subdued title
action → user-triggerable (button, chip, toggle) — 600 weight label, stretched in primary-task
feedback → snackbar / inline confirmation — opacity 0.90, italic (transient)
context → supporting info (weather next to clock, related card) — opacity 0.82, smaller card padding (14px)
navigation → tabs, bottom-nav, back arrow — neutral (opacity 0.96)
Group Role → Surface Treatment 5 roles
chrome → device frame zones (topSystem / bottomNav) — absolute-positioned, no group wrapper
primary-task → main task unit — unified inner surface (28px radius, gradient, drop shadow, 20px padding)
supporting → context that frames the primary — dashed border, indented (8px margin), opacity 0.90
tertiary → low importance (badges, quiet notifications) — minimal, opacity 0.72
meta → overflow / settings shortcuts — subtle background, opacity 0.84
Component role + group role together drive css/genui.css's role-aware styling. Set by the composer LLM, validated by validateLayout().
Governance Triggers → Auto Additions planned
explanation_needed → auto-append "Why this?" link to layout bottom
override_needed → auto-append "Not now" button to action-row
handoff_required → device badge 📱→⌚ on relevant card
high_impact_decision → confirm scrim + biometric prompt gate
These rules are declared but not yet auto-applied to the canvas. Edit affordance coming in R4.
Pipeline Atomic Bridge 21 mappings
Path A's selector emits componentIds directly from component_registry.json. The renderer (app/scenes.js) routes specific IDs through Path B's renderAtomicForRole() for richer One UI rendering. Defined in PIPELINE_CHROME_ATOMIC_ROLE + PIPELINE_BODY_ATOMIC_ROLE.
CHROME BRIDGE — 6 mappings (zone-positioned)
container.status-bar-app → status-bar
status-bar.default → status-bar
container.header → collapsed-app-bar
container.nav-gestures-dark → gesture-bar
container.nav-buttons-light → nav-buttons
dialog.nav-gesture-bar → gesture-bar
BODY BRIDGE — 15 mappings (inline)
input_summary_card → focus-block (kind=secondary if value > 28 chars)
weather_glance_card → focus-block
calendar_summary_card → focus-block
message_summary_card → focus-block
eta_card → focus-block
reminder_card → focus-block
media_control_bar → now-bar (variant inferred — media)
now-bar.media-player → now-bar (media)
now-bar.charging → now-bar (charging)
navigation_turn_card → now-bar (single-line / voice)
action_chip_row → action-row
quick_toggle_row → toggle-chip
notification-card → notif-card
notification.ai-regular → notif-card-ai
lock-screen.clock → clock
Components NOT in this bridge render via their own template in app/templates.js (or fall through to the registry-fallback stub).
