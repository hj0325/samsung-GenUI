Foundation

Motion Principles

One UI motion is guided by four principles that mirror its design philosophy: Natural, Responsive, Contextual, and Delightful. Animations should feel organic and purposeful, never decorative.
Natural
Motion follows real-world physics. Objects accelerate and decelerate like physical things. One UI avoids linear timing — everything has momentum.
Decelerate curves for enters, accelerate for exits
Responsive
Immediate feedback on touch. Ripples start at the point of contact. State changes acknowledge user input within 100ms.
Touch feedback < 100ms | Visual response < 200ms
Contextual
Direction matters. Elements enter from where they came from and exit toward where they're going. Spatial consistency builds mental models.
Shared axis transforms maintain spatial continuity
Delightful
Subtle spring physics and overshoot in key moments create warmth. One UI 7 emphasizes organic, sentimental interactions.
Spring easing for FAB, dialogs, toggles | Bounce for celebration

Samsung One UI Basic Path Interpolator

One UI Basic Path System Default
cubic-bezier(0.22, 0.25, 0.00, 1.00)
The foundational curve for all One UI system transitions. Accelerates quickly then slows down gradually. All other curves derive from or complement this base. Used by the OS for app open/close, page transitions, and system-level animations.

Supplementary Motion Styles (Samsung Official)

Four additional easing approaches from Samsung developer documentation, used alongside the derived curves for specific contexts.
Linear
linear
Consistent velocity throughout. For transparency shifts, screen dissolves.
Ease In
ease-in
Starts slow, accelerates. When origin is visible but endpoint undefined (exits).
Ease Out
ease-out
Starts fast, decelerates. When speed reduction needs visibility. Avoid for off-screen exits.
Ease In Out
ease-in-out
Slow start, accelerates midway, decelerates at end. For highly visible motion sequences.

09 / Patterns

Animation Patterns

Enter vs Exit Asymmetry
Entrances are slower and more dramatic than exits. This focuses user attention on what's arriving, not what's leaving.
Enter: 300ms
Exit: 200ms
Stagger Delay
Sequential items enter with a 50-80ms delay between each, creating a cascade effect. Keeps total sequence under 500ms.
60ms stagger | max 5 items | total < 500ms
Spatial Continuity
Elements exit toward their destination and enter from their origin. A list item expanding to detail view grows from the item's position.
Container transforms preserve spatial relationships
Reduce Motion
When the user enables "Reduce animations" in accessibility settings, replace all motion with instant opacity transitions (100ms fade).
@media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; } }
