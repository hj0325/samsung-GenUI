# Evolve — Design Refinement Knowledge Base

## Purpose
This document accumulates design issues discovered through iterative refinement.
Each entry captures a violated constraint, the fix applied, and the lesson learned.
When generating new UI, these lessons are extracted as additional constraints to prevent recurring issues.

---

## Format

Each entry follows this structure:

```
### E{number}: {title}
- **Type**: spacing | hierarchy | density | alignment | sizing | readability | consistency | semantic | interaction
- **Severity**: high | medium | low
- **Scenario**: {which screen type}
- **Issue**: {what was wrong}
- **Fix**: {what was changed}
- **Constraint**: {the rule to prevent this in future generation}
- **Date**: {YYYY-MM-DD}
```
